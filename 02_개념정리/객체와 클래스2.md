# 객체와 클래스

## 핵심 요약

클래스는 객체를 만들기 위한 설계도이고, 객체는 그 설계도로 실제 생성된 결과물이다.

```text
클래스 = 설계도
객체 = 설계도로 만든 실제 물건
```

---

## 예시

```python
class Television:
    pass

tv1 = Television()
tv2 = Television()
```

`Television`은 클래스이고, `tv1`, `tv2`는 객체이다.

---

## 왜 쓰는가?

변수와 함수를 따로따로 두면 코드가 커질수록 관리가 어렵다.  
클래스를 쓰면 관련 있는 데이터와 기능을 하나로 묶을 수 있다.

```text
TV의 데이터: 채널, 볼륨, 전원 상태
TV의 기능: 채널 변경, 볼륨 변경, 전원 켜기/끄기
```

이것을 하나로 묶으면 `Television` 클래스가 된다.

---

## 코드에서 나온 예시

```python
tv1 = Television(1, 10, True)
tv2 = Television(2, 20, False)
tv3 = Television(3, 30, True)
```

같은 클래스로 여러 객체를 만들 수 있다.

---

## 내 식으로 이해

클래스는 붕어빵 틀이고, 객체는 실제 붕어빵이다.  
틀은 하나여도 붕어빵은 여러 개 만들 수 있다.

`Television`이라는 틀 하나로 `tv1`, `tv2`, `tv3`를 만들 수 있는 것과 같다.
