## 한 줄 정의

튜플, 세트, 딕셔너리는 리스트와 함께 Python에서 자주 사용하는 자료구조이다.

|자료구조|특징|
|---|---|
|튜플|순서가 있고 변경 불가능|
|세트|중복이 없고 순서가 없음|
|딕셔너리|key와 value로 저장|

## 핵심 개념

- 시퀀스
    
- 튜플
    
- 패킹과 언패킹
    
- `enumerate()`
    
- 세트
    
- 집합 연산
    
- 딕셔너리
    
- key-value
    
- 딕셔너리 메서드
    
- 문자열 메서드
    

## 1. 시퀀스

시퀀스는 순서가 있는 자료구조이다.

대표적인 시퀀스:

```text
문자열
리스트
튜플
range
```

시퀀스는 인덱싱과 슬라이싱이 가능하다.

```python
s = "Python"

print(s[0])
print(s[1:4])
```

## 2. 튜플

튜플은 리스트와 비슷하지만 변경할 수 없다.

```python
fruits = ("apple", "banana", "grape")

print(fruits[0])
```

튜플은 소괄호 `()`를 사용한다.

## 3. 튜플의 특징

튜플은 한 번 만들면 요소를 바꿀 수 없다.

```python
fruits = ("apple", "banana", "grape")

fruits[1] = "pear"
```

이 코드는 오류가 난다.

```text
TypeError
```

## 4. 요소가 하나인 튜플

요소가 하나인 튜플은 쉼표가 필요하다.

```python
single_tuple = ("apple",)
```

쉼표가 없으면 튜플이 아니라 그냥 문자열로 인식된다.

```python
no_tuple = ("apple")
```

## 5. 리스트와 튜플 변환

리스트를 튜플로 바꿀 수 있다.

```python
myList = [1, 2, 3, 4]
myTuple = tuple(myList)

print(myTuple)
```

튜플을 리스트로 바꿀 수 있다.

```python
myTuple = (1, 2, 3, 4)
myList = list(myTuple)

print(myList)
```

## 6. 튜플 패킹과 언패킹

여러 값을 하나로 묶는 것을 패킹이라고 한다.

```python
t = 10, 20, 30
```

튜플 값을 여러 변수에 나누어 담는 것을 언패킹이라고 한다.

```python
a, b, c = t
```

값을 서로 바꿀 때도 사용할 수 있다.

```python
x = 10
y = 20

x, y = y, x
```

## 7. 함수에서 여러 값 반환

Python 함수가 여러 값을 반환할 때 튜플을 사용한다.

```python
def get_info():
    name = "Kim"
    age = 20
    return name, age

student_name, student_age = get_info()
```

## 8. `enumerate()`

리스트를 반복하면서 인덱스도 함께 얻을 수 있다.

```python
fruits = ["apple", "banana", "grape"]

for index, value in enumerate(fruits):
    print(index, value)
```

결과:

```text
0 apple
1 banana
2 grape
```

## 9. 세트

세트는 중복을 허용하지 않는 자료구조이다.

```python
numbers = set([1, 2, 3, 1, 2, 3])

print(numbers)
```

결과:

```text
{1, 2, 3}
```

## 10. 세트의 특징

```text
중복이 없다.
순서가 없다.
인덱스로 접근할 수 없다.
집합 연산이 가능하다.
```

공백 세트는 `{}`가 아니라 `set()`으로 만든다.

```python
empty_set = set()
```

`{}`는 공백 딕셔너리이다.

## 11. 세트에 값 추가

```python
fruits = {"apple", "banana"}

fruits.add("grape")

print(fruits)
```

## 12. 세트에서 값 삭제

```python
fruits.remove("apple")
```

`remove()`는 값이 없으면 오류가 난다.

오류 없이 삭제하려면 `discard()`를 사용할 수 있다.

```python
fruits.discard("apple")
```

## 13. 세트 집합 연산

```python
A = {1, 2, 3, 4, 5}
B = {3, 4, 5, 6, 7}
```

|연산|코드|결과|
|---|---|---|
|교집합|`A & B`|`{3, 4, 5}`|
|합집합|`A|B`|
|차집합|`A - B`|`{1,2}`|
|대칭 차집합|`A ^ B`|`{1,2,6,7}`|

## 14. 리스트에서 중복 제거

```python
list1 = [1, 2, 3, 4, 5, 1, 2, 4]

unique = set(list1)

print(unique)
print(len(unique))
```

서로 다른 값의 개수를 구할 수 있다.

## 15. 딕셔너리

딕셔너리는 key와 value로 데이터를 저장한다.

```python
capitals = {
    "Korea": "Seoul",
    "USA": "Washington",
    "UK": "London"
}
```

값을 꺼낼 때는 key를 사용한다.

```python
print(capitals["Korea"])
```

## 16. 딕셔너리 특징

```text
key와 value로 저장한다.
key는 중복될 수 없다.
value는 중복될 수 있다.
key를 이용해 값을 빠르게 찾을 수 있다.
```

## 17. 딕셔너리 생성

공백 딕셔너리:

```python
capitals = {}
```

값 추가:

```python
capitals["Korea"] = "Seoul"
capitals["USA"] = "Washington"
```

## 18. 딕셔너리 값 가져오기

```python
print(capitals["Korea"])
```

없는 key를 사용하면 오류가 난다.

```python
print(capitals["France"])
```

오류를 피하려면 `get()`을 사용한다.

```python
print(capitals.get("France", "해당 키가 없습니다."))
```

## 19. 딕셔너리 삭제

```python
city = capitals.pop("UK")
```

없는 key를 삭제하면 오류가 날 수 있으므로 확인 후 삭제한다.

```python
if "UK" in capitals:
    capitals.pop("UK")
```

## 20. 딕셔너리 반복

key만 반복:

```python
for key in capitals:
    print(key, capitals[key])
```

key와 value를 함께 반복:

```python
for key, value in capitals.items():
    print(key, value)
```

## 21. 딕셔너리 메서드

```python
capitals.keys()
capitals.values()
capitals.items()
```

|메서드|의미|
|---|---|
|`keys()`|key 목록|
|`values()`|value 목록|
|`items()`|key-value 쌍|

## 22. 딕셔너리 함축

```python
values = [1, 2, 3, 4, 5, 6]

dic = {x: x ** 2 for x in values if x % 2 == 0}

print(dic)
```

결과:

```text
{2: 4, 4: 16, 6: 36}
```

## 23. 문자열도 시퀀스

문자열도 인덱싱과 슬라이싱이 가능하다.

```python
s = "Monty Python"

print(s[0])
print(s[-1])
print(s[6:10])
```

문자열은 변경할 수 없는 불변 객체이다.

```python
word = "abcdef"
word[0] = "A"
```

이 코드는 오류가 난다.

새 문자열을 만들어야 한다.

```python
word = "A" + word[1:]
```

## 24. 문자열 메서드

```python
s = "i am a student."

print(s.capitalize())
print(s.upper())
print(s.lower())
```

문자열 찾기와 바꾸기:

```python
s = "www.naver.com"

print(s.replace("com", "co.kr"))
print(s.find(".com"))
print(s.count("."))
```

문자열 분리:

```python
s = "Welcome to Python"

words = s.split()

print(words)
```

문자열 결합:

```python
result = ",".join(["apple", "grape", "banana"])

print(result)
```

## 대표 예제: 연락처 관리

```python
def get_contact():
    name = input("이름: ")
    number = input("전화번호: ")
    return name, number

def display_menu():
    print("1. 연락처 추가")
    print("2. 연락처 삭제")
    print("3. 연락처 검색")
    print("4. 연락처 출력")
    print("5. 종료")

    select = int(input("메뉴 항목을 선택하시오: "))
    return select

def main():
    address_book = {}

    while True:
        user = display_menu()

        if user == 1:
            name, number = get_contact()
            address_book[name] = number
        elif user == 2:
            name, number = get_contact()

            if name in address_book:
                address_book.pop(name)
        elif user == 3:
            name = input("검색할 이름: ")
            print(address_book.get(name, "연락처가 없습니다."))
        elif user == 4:
            for key in sorted(address_book):
                print(key, "의 전화번호:", address_book[key])
        else:
            break

main()
```

## 대표 예제: 단어 카운터

```python
text_data = "Create the highest grandest vision possible for your life because you become what you believe"

word_dic = {}

for word in text_data.split():
    if word in word_dic:
        word_dic[word] += 1
    else:
        word_dic[word] = 1

for word, count in sorted(word_dic.items()):
    print(word, "의 등장횟수 =", count)
```

## 오늘의 핵심 정리

```text
튜플은 변경할 수 없는 순서 있는 자료구조이다.
세트는 중복을 허용하지 않는 자료구조이다.
딕셔너리는 key와 value로 저장하는 자료구조이다.
세트는 교집합, 합집합, 차집합 연산이 가능하다.
딕셔너리는 key로 value를 찾는다.
문자열도 인덱싱과 슬라이싱이 가능하다.
문자열은 불변 객체이다.
```

## 헷갈린 점 정리

### 리스트와 튜플

|구분|리스트|튜플|
|---|---|---|
|기호|`[]`|`()`|
|변경 가능|가능|불가능|

### 리스트와 세트

|구분|리스트|세트|
|---|---|---|
|순서|있음|없음|
|중복|허용|허용 안 함|
|인덱스|가능|불가능|

### 딕셔너리

```python
student = {
    "name": "Kim",
    "score": 90
}
```

`name`과 `score`는 key이고, `"Kim"`과 `90`은 value이다.

## 내 식으로 이해

리스트가 여러 값을 순서대로 저장하는 구조라면, 튜플은 바뀌면 안 되는 값을 묶는 구조이고, 세트는 중복 제거와 집합 연산에 유리한 구조이며, 딕셔너리는 이름표를 붙여 값을 찾는 구조이다.

즉, 자료의 성격에 따라 자료구조를 다르게 선택해야 한다.
