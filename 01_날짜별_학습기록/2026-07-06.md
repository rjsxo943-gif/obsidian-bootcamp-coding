## 📚 배운 개념

## 1. 객체와 클래스

클래스는 객체를 만들기 위한 설계도이다.  
객체는 그 설계도로 실제 만들어진 결과물이다.

```python
tv1 = Television(1, 10, True)
tv2 = Television(2, 20, False)
```

같은 `Television` 클래스로 만들었지만 `tv1`, `tv2`는 서로 다른 객체이다.

---

## 2. 생성자와 self

`__init__`은 객체가 만들어질 때 자동으로 실행된다.

```python
def __init__(self, channel, volume, on):
    self.channel = channel
    self.volume = volume
    self.on = on
```

`self`는 지금 만들어진 객체 자신을 뜻한다.  
즉 `self.channel`은 각 객체가 자기 채널 값을 따로 가진다는 의미다.

---

## 3. 클래스 변수와 인스턴스 변수

```python
serial_number = 0
```

클래스 변수는 같은 클래스로 만든 객체들이 공유한다.

```python
self.channel = channel
self.volume = volume
self.on = on
```

인스턴스 변수는 객체마다 따로 가진다.

---

## 4. 특수 메소드

```python
def __str__(self):
    return f"Television(channel={self.channel}, volume={self.volume}, on={self.on})"
```

`__str__`을 정의하면 객체를 `print()`로 출력할 때 사람이 보기 좋은 문자열로 바꿀 수 있다.

```python
def __add__(self, other):
    new_width = self.width + other.width
    new_height = self.height + other.height
    return Rectangle(new_width, new_height)
```

`__add__`를 정의하면 객체끼리 `+` 연산을 할 수 있다.

---

## 5. tkinter GUI

`tkinter`는 Python 기본 GUI 라이브러리이다.

```python
root = tk.Tk()
root.title("메모장")
root.geometry("800x600")
root.mainloop()
```

GUI 프로그램은 창을 만들고, 위젯을 배치하고, 이벤트가 발생하면 함수가 실행되는 구조이다.

---

## 6. 파일 열기와 저장

```python
with open(path, "w", encoding="utf-8") as f:
    f.write(self.text.get("1.0", tk.END))
```

파일 저장은 `open()`으로 파일을 열고, 텍스트 위젯의 내용을 가져와서 쓰는 구조이다.  
한글 저장을 위해 `encoding="utf-8"`이 중요하다.

---

## 7. Canvas와 마우스 이벤트

```python
self.canvas.bind("<Button-1>", self.start_draw)
self.canvas.bind("<B1-Motion>", self.draw)
self.canvas.bind("<ButtonRelease-1>", self.stop_draw)
```

마우스를 누르고 움직이고 떼는 동작을 각각 함수와 연결했다.  
이벤트가 발생하면 연결된 콜백 함수가 실행된다.

## 8. NumPy 배열

```python
ftemp = [32, 63, 73, 82, 91]
F = np.array(ftemp)
```

Python 리스트를 NumPy 배열로 바꾸면 이후 데이터 분석, 행렬 계산, AI 연산에서 더 편하게 사용할 수 있다.

# 핵심 코드

```python
class Television:
    serial_number = 0

    def __init__(self, channel, volume, on):
        Television.serial_number += 1
        self.serial_number = Television.serial_number
        self.channel = channel
        self.volume = volume
        self.on = on

    def __str__(self):
        return f"Television(channel={self.channel}, volume={self.volume}, on={self.on})"
```

self.canvas.bind("<Button-1>", self.start_draw)

## ⚠️ 오늘 헷갈린 부분

- 클래스 변수와 인스턴스 변수의 차이
- `self`를 왜 매개변수로 넣는지
- 객체를 출력하면 원래 이상한 주소처럼 나오는데 `__str__`을 쓰면 왜 문자열이 나오는지
- `__add__`를 만들면 객체끼리 더하기가 가능해지는 원리
- GUI에서 버튼을 누르면 왜 함수가 실행되는지
- `bind()`에서 이벤트 문자열과 콜백 함수가 어떻게 연결되는지
- 메모장 코드에서 `Text` 위젯의 `"1.0"`과 `tk.END`가 의미하는 범위
- 그림판의 지우개가 실제 삭제가 아니라 흰색으로 덮어 그리는 방식이라는 점
- Python 리스트와 NumPy 배열이 겉보기에는 비슷하지만 내부 목적이 다르다는 점

## 🧠 내 식으로 이해

오늘 배운 내용은 결국 **데이터와 기능을 하나로 묶는 객체지향 방식으로 프로그램을 구조화하고, 그 구조를 이용해 실제 창이 있는 프로그램까지 만들어 본 것**이다.

객체지향은 그냥 어려운 문법이 아니라, `TV`, `자동차`, `원`, `사각형`, `주사위`처럼 현실의 대상을 코드 안에 하나의 덩어리로 만드는 방법이다.

`tkinter`는 그 객체지향 구조를 실제 화면 프로그램에 적용하는 연습이었다.  
메모장은 글을 저장하는 프로그램이고, 그림판은 마우스 이벤트를 받아서 선을 그리는 프로그램이다.

NumPy는 나중에 AI나 데이터 분석을 할 때 리스트보다 더 계산에 적합한 배열을 쓰기 위한 출발점이다.
