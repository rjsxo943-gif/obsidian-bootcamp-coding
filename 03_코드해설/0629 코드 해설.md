# 1. `34.py` 코드 해설

## 원본 코드 정리

```python
treeHit = 0

while treeHit < 10:
    treeHit += 1
    print("나무를 %d번 찍었습니다." % treeHit)

    if treeHit == 10:
        print("나무 넘어갑니다.")

print("나무를 %d번 찍었습니다." % treeHit)
```

---

## 코드 목적

`while`문을 사용해서 나무를 10번 찍을 때까지 반복하고, 10번이 되면 나무가 넘어간다는 문장을 출력하는 코드이다.

---

## 코드 흐름

```text
1. treeHit을 0으로 시작
2. treeHit이 10보다 작으면 반복
3. 반복할 때마다 treeHit 1 증가
4. 현재 몇 번 찍었는지 출력
5. treeHit이 10이 되면 "나무 넘어갑니다." 출력
6. while문 종료 후 마지막 print 실행
```

---

## 핵심 코드 1: `while` 조건

```python
while treeHit < 10:
```

`treeHit`이 10보다 작을 동안 반복한다.

처음에는 `treeHit = 0`이므로 조건이 참이다.

---

## 핵심 코드 2: 카운트 증가

```python
treeHit += 1
```

이 코드는 다음과 같다.

```python
treeHit = treeHit + 1
```

반복할 때마다 나무를 찍은 횟수가 1씩 증가한다.

---

## 핵심 코드 3: 조건문

```python
if treeHit == 10:
    print("나무 넘어갑니다.")
```

나무를 10번 찍으면 문장을 출력한다.

---

## 주의할 점: 들여쓰기

마지막 줄은 `while`문 밖에 있다.

```python
print("나무를 %d번 찍었습니다." % treeHit)
```

그래서 반복문이 끝난 뒤 한 번 더 출력된다.

이 부분이 의도한 것인지 확인해야 한다.

만약 반복문 안에서만 출력하고 싶다면 마지막 `print()`는 제거해도 된다.

## 2-1. 커피 자판기 예제

```python
coffee = 10

while True:
    money = int(input("돈을 넣어 주세요: "))

    if money == 300:
        print("커피를 줍니다.")
        coffee = coffee - 1
    elif money > 300:
        print(f"거스름돈 {money - 300}를 주고 커피를 줍니다.")
        coffee = coffee - 1
    else:
        print("돈을 다시 돌려주고 커피를 주지 않습니다.")

    print(f"남은 커피의 양은 {coffee}개입니다.")

    if coffee == 0:
        print("커피가 다 떨어졌습니다. 판매를 중지 합니다.")
        break
```

### 코드 목적

돈을 입력받아 커피를 줄지 말지 판단하고, 커피가 다 떨어지면 반복을 종료하는 코드이다.

### 핵심 개념

```text
while True: 무한 반복
if / elif / else: 돈의 조건에 따른 분기
break: 반복문 종료
```

### 코드 흐름

```text
1. 커피 개수를 10개로 시작
2. 돈을 입력받음
3. 300원이면 커피 제공
4. 300원보다 많으면 거스름돈과 커피 제공
5. 300원보다 적으면 돈을 돌려주고 커피 제공 안 함
6. 커피가 0개가 되면 break로 반복 종료
```

### 주의할 점

```python
money = int(input("돈을 넣어 주세요: "))
```

아무것도 입력하지 않고 엔터를 치면 빈 문자열 `""`을 정수로 바꾸려고 하므로 `ValueError`가 발생할 수 있다.

나중에는 예외 처리를 사용하면 더 안전하다.

```python
try:
    money = int(input("돈을 넣어 주세요: "))
except ValueError:
    print("숫자를 입력하세요.")
```

## 2-2. `continue` 예제

```python
a = 0

while a < 10:
    a = a + 1

    if a % 2 == 0:
        continue

    print(a)
```

### 코드 목적

### 코드 해석

그래서 출력 결과는 홀수만 나온다.

```text

7
```

|키워드|의미|
|---|---|
|`break`|반복문 자체를 끝냄|
|`continue`|이번 반복만 건너뛰고 다음 반복으로 감|

## 3-1. `match-case` 계산기

```python
match choice:
    case "1":
        print(f"{num1} + {num2} = {add(num1, num2)}")
    case "2":
        print(f"{num1} - {num2} = {subtract(num1, num2)}")
    case "3":
        print(f"{num1} * {num2} = {multiply(num1, num2)}")
    case "4":
        result = divide(num1, num2)
        print(f"{num1} / {num2} = {result}")
    case _:
        print("Invalid input")
```

### 코드 해석

`match-case`는 여러 경우 중 하나를 선택하는 문법이다.

기존의 `if-elif-else`와 비슷한 역할을 한다.

`case _`는 위 조건에 모두 해당하지 않을 때 실행되는 기본값이다.

## 3-3. 선택에 따라 더하기/곱하기

```python
def add_mul(choice, *args):
    if choice == "add":
        result = 0

        for i in args:
            result = result + i

    elif choice == "mul":
        result = 1

        for i in args:
            result = result * i

    return result
```

### 코드 해석

첫 번째 인수 `choice`에 따라 동작이 달라진다.

```python
add_mul("add", 1, 2, 3)
```

이면 더하기를 수행한다.

```python
add_mul("mul", 1, 2, 3, 4, 5)
```

이면 곱하기를 수행한다.

곱셈에서 `result = 1`로 시작하는 이유는 곱셈의 항등원이 1이기 때문이다.

## 3-4. `**kwargs`

```python
def print_kwargs(**kwargs):

```

### 코드 해석

`**kwargs`는 키워드 인수를 딕셔너리 형태로 받는다.

예를 들어 다음처럼 호출하면,

```python

def add(a, b):
```

# 9. `avg_args.py` 코드 해설

## 원본 코드 정리

```python
import sys

args = sys.argv[1:]

if len(args) == 0:
    print("숫자를 하나 이상 입력하세요.")
else:
    total = 0

    for i in args:
        total += int(i)

    avg = total / len(args)

    print("합계:", total)
    print("평균:", f"{avg:.2f}")
```

# ✅ 오늘의 핵심 정리

```text
while문은 조건이 참인 동안 반복한다.
while True는 무한 반복 구조이다.
break는 반복문을 종료한다.
continue는 이번 반복을 건너뛰고 다음 반복으로 간다.
for문은 리스트나 range()를 순회할 때 사용한다.
range(1, 101)은 1부터 100까지이다.
중첩 반복문은 반복문 안에 반복문이 있는 구조이다.
별 찍기는 반복문의 구조를 시각적으로 이해하는 연습이다.
함수는 코드를 기능 단위로 묶는 방법이다.
return은 함수 결과를 밖으로 돌려준다.
*args는 여러 위치 인수를 받는다.
**kwargs는 여러 키워드 인수를 딕셔너리로 받는다.
global은 함수 안에서 전역 변수를 직접 수정하겠다는 의미이다.
독스트링은 함수 설명문이다.
sys.argv는 터미널에서 입력한 명령행 인자를 받는다.
sys.argv로 받은 값은 문자열이므로 숫자 계산에는 int() 변환이 필요하다.
파일 입출력에서는 open(), close(), with, encoding 개념이 중요하다.
```
